Secret `homepage-widgets` (namespace `dashboard`) — keys must match `values.yaml` `extraEnv` / ingress widget refs.

Keys (stringData):

  qbit-user       qBittorrent Web UI username
  qbit-pass       qBittorrent Web UI password
  sonarr-apikey   Sonarr → Settings → General → Security → API Key
  radarr-apikey   Radarr → Settings → General → Security → API Key

1) Quick test on cluster (not for Git)

kubectl create secret generic homepage-widgets -n dashboard \
  --from-literal=qbit-user='REPLACE' \
  --from-literal=qbit-pass='REPLACE' \
  --from-literal=sonarr-apikey='REPLACE' \
  --from-literal=radarr-apikey='REPLACE'

kubectl rollout restart deployment -n dashboard -l app.kubernetes.io/name=homepage

2) Sealed Secret (kubeseal + controller running)

kubectl create secret generic homepage-widgets -n dashboard \
  --dry-run=client -o yaml \
  --from-literal=qbit-user='REPLACE' \
  --from-literal=qbit-pass='REPLACE' \
  --from-literal=sonarr-apikey='REPLACE' \
  --from-literal=radarr-apikey='REPLACE' \
  | kubeseal -o yaml > homepage-widgets.sealed.yaml

kubectl apply -f homepage-widgets.sealed.yaml

Plain `secrets/*.yaml` is gitignored; commit only `.sealed.yaml` if that fits your workflow.
